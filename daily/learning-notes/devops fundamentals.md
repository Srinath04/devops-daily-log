### **DevOps Fundamentals Reinforce**

- **DevOps fundamentals**
    
    - DevOps as a **culture + process**, more than tools
        
    - Focus on collaboration between **Dev, QA, Ops**
        
    - Speed, reliability, automation, feedback loops
        
- **SDLC (Software Development Life Cycle)**
    
    - Requirement → Design → Development → Testing → Deployment → Maintenance
        
    - Where DevOps plugs in: **CI/CD, automation, monitoring, fast feedback**

- **Working models**
    
    - Waterfall vs Agile (Scrum/Kanban)
        
    - Agile = iterative delivery, frequent releases, adaptability
        
    - DevOps thrives best with **Agile workflows**
        
- **How tasks are usually formulated**
    
    - Business / Client → Product Owner / Manager
        
    - Converted into **requirements / tickets**
        
    - Prioritized based on impact, urgency, dependencies
        
    - Assigned to Dev / Ops / DevOps based on responsibility
        
- **JIRA basics**
    
    - Tickets as the **single source of truth**
        
    - Issue types: Task, Bug, Story, Epic
        
    - Status flow: To Do → In Progress → Review → Done
        
    - Used for tracking, accountability, and sprint planning
